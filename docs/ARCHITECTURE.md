## 數據流程

### 1. 標案採集流程

```
定時任務 (GitHub Actions)
        ↓
   啟動爬蟲腳本
        ↓
┌───────────────────────┐
│  數據來源              │
│  • 政府採購網          │
│  • 企業採購平台        │
│  • Google Alerts       │
└───────────────────────┘
        ↓
   數據清洗與標準化
        ↓
   存入 tenders 表
        ↓
   觸發 AI 分析
        ↓
┌───────────────────────┐
│  AI 分析引擎            │
│  • 標案評分            │
│  • 報價建議            │
│  • 風險評估            │
│  • 投標策略            │
└───────────────────────┘
        ↓
   更新 AI 分析結果
        ↓
   準備推送
```

---

### 2. 每日推送流程

```
定時任務 (每天 8:00)
        ↓
   查詢今日新標案
        ↓
   篩選符合條件的標案
        ↓
   生成 AI 分析報告
        ↓
   格式化為 Telegram 訊息
        ↓
   通過 Telegram Bot API 發送
        ↓
   記錄到 notifications 表
```

---

### 3. 用戶投標流程

```
用戶瀏覽標案列表
        ↓
   點擊查看詳情
        ↓
   查看 AI 分析與建議
        ↓
   點擊「立即投標」
        ↓
   填寫投標資訊
        ↓
   生成投標文件
        ↓
   存入 bids 表
        ↓
   更新投標狀態
        ↓
   設定跟進提醒
```

## API 路由架構

```
/api
├── /auth
│   ├── POST /login
│   ├── POST /logout
│   └── GET  /me
│
├── /dashboard
│   └── GET  /
│
├── /tenders
│   ├── GET    /              # 列表
│   ├── GET    /:id           # 詳情
│   ├── POST   /:id/bid       # 投標
│   ├── PUT    /:id/watch     # 追蹤
│   └── DELETE /:id           # 刪除
│
├── /clients
│   ├── GET    /              # 列表
│   ├── GET    /:id           # 詳情
│   ├── POST   /              # 新增
│   ├── PUT    /:id           # 更新
│   └── DELETE /:id           # 刪除
│
├── /clients/:client_id
│   ├── /contacts
│   │   ├── GET    /          # 聯絡人列表
│   │   ├── POST   /          # 新增聯絡人
│   │   └── /:contact_id
│   │       ├── GET    /      # 聯絡人詳情
│   │       ├── PUT    /      # 更新
│   │       └── DELETE /      # 刪除
│   │
│   └── /interactions
│       ├── GET    /          # 互動記錄列表
│       └── POST   /          # 新增互動記錄
│
├── /bids
│   ├── GET    /              # 列表
│   ├── GET    /:id           # 詳情
│   ├── PUT    /:id           # 更新
│   └── DELETE /:id           # 刪除
│
├── /reports
│   ├── GET /bid-statistics   # 投標統計
│   ├── GET /profit-analysis  # 利潤分析
│   └── GET /industry-analysis# 產業分析
│
├── /settings
│   ├── GET  /                # 獲取設定
│   └── PUT  /                # 更新設定
│
└── /notifications
    ├── GET  /                # 通知列表
    └── POST /test            # 測試通知
```
